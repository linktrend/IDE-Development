# Plan Writing

Use this skill when a task needs structured planning before implementation.

## Output Shape

- objective
- current state
- constraints
- assumptions
- task breakdown
- dependencies
- verification
- open risks or blockers

## Rules

1. Break work into verifiable steps.
2. Prefer concrete outputs over abstract categories.
3. Separate confirmed facts from assumptions.
4. Keep plans short enough to execute, not just admire.
5. Reuse existing templates when a local template fits the task.
