# Output Contract

## Purpose

This document defines the outputs each work unit or stage must produce before a valid handoff can occur.

## Contract Responsibilities

The output contract must define:

- required artifacts
- required evidence
- required decisions
- required completion statements

## Universal Output Rule

A stage is not contract-complete merely because activity occurred. It is complete only when required outputs exist and are consumable by the next stage.

## Work Unit Output Contracts

### Intent

Required outputs:

- intent artifact
- verdict
- eligibility decision for program creation

### Program

Required outputs:

- program artifact
- module structure
- program definition of done
- global constraints

### Module

Required outputs:

- module artifact
- phase structure
- issue decomposition or clear issue references
- module definition of done

### Phase

Required outputs:

- phase artifact
- issue list
- phase completion criteria

### Issue

Required outputs:

- executed work or explicit blocker trail
- issue state progression
- proof handoff eligibility

### Proof

Required outputs:

- evidence mapped to criteria
- referenced artifacts or observations
- failure and gap visibility where relevant

### Review

Required outputs:

- review artifact
- verdict
- findings
- next action

### Integration

Required outputs:

- integration artifact
- final state
- downstream readiness implications

## Ownership Rule

The producing stage owns output completeness.

The consuming stage owns acceptance of output sufficiency.
