# Discovery

This directory defines the optional discovery and interview layer used before `INTENT.md` in greenfield or ambiguous work.

Discovery is not part of routine execution.

Use it only when:

- the project is greenfield
- the request is ambiguous
- the user goal is under-specified
- you are about to guess missing requirements silently

Do not use it for:

- bug fixes
- existing modules with clear scope
- routine work
- small self-contained features

Read `INDEX.yaml` first.

Typical read path:

1. `DISCOVERY.md`
2. `INTERVIEW.md`
3. `QUESTIONING-GUIDELINES.md`
4. `CONFIDENCE-THRESHOLDS.md`
5. `ANTI-RATIONALIZATION.md`
6. `INTENT.md`

Discovery feeds `INTENT.md`. It does not bypass or replace intent formation.
