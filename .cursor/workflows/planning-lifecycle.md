# Planning Lifecycle

## Purpose

Turn an ambiguous request into an execution-ready plan without starting implementation prematurely.

## Stages

1. clarify the objective
2. define the finished state
3. capture constraints and assumptions
4. create program and issue structure
5. verify internal consistency
6. hand off to execution

## Deliverables

- intent
- program plan
- initial issues
- verification checklist

## Guardrails

- avoid over-design
- avoid parallel artifacts that say the same thing
- do not mark work ready until acceptance criteria and proof are clear
