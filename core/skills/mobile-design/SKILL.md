# Mobile Design

Use this skill when mobile or narrow viewport experience matters.

## Checklist

- primary actions are reachable
- tap targets are large enough
- content does not overflow
- navigation is clear
- forms are ergonomic
- keyboard appearance is considered
- loading/error/empty states work on small screens
- platform conventions are respected when relevant

## Rules

- Design mobile layouts deliberately, not as shrunken desktop.
- Avoid hover-only interactions.
- Do not hide critical actions behind unclear gestures.
- Verify with mobile viewport screenshots when possible.
- Keep text readable without viewport-based font scaling.

## Proof

Useful evidence includes:

- mobile viewport screenshot
- tested screen width
- interaction notes
- accessibility/touch notes
- known limitations

## Progressive Disclosure

Read the active issue, responsive styles, affected components, and existing mobile patterns. Do not redesign unrelated flows.
