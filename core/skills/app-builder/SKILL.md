# App Builder

Use this skill when creating a new app surface or extending an existing app across multiple layers.

## Use When

- scaffolding a new project or app area
- building a full-stack feature
- selecting from existing templates
- coordinating frontend, backend, data, and tests

## Rules

- Start from `INTENT`, `PROGRAM`, `MODULE`, or `ISSUE` artifacts as appropriate.
- Prefer existing stack and templates.
- Decompose into issue-sized slices before coding.
- Do not invent architecture when repository conventions exist.
- Verify each vertical slice before expanding.

## Build Checklist

- project type and stack confirmed
- target users and core flow known
- data/API boundaries defined
- UI states identified
- tests and validation plan defined
- deployment/run instructions known

## Output

- selected stack or template
- implementation slices
- created or changed files
- verification evidence
- blockers or open decisions

## Progressive Disclosure

Read the active artifact, repo structure, package scripts, and nearest similar app area. Do not scan unrelated apps unless choosing a template requires comparison.
