# Workflows

Workflows describe multi-step operating patterns that span doctrine, artifacts, runtime behavior, prompts, templates, and checklists.

Read `INDEX.yaml` first.

Preferred workflow specifications for new work are:

- `WORKFLOW-MODEL.md`
- `PROGRAM-WORKFLOW.md`
- `MODULE-WORKFLOW.md`
- `ISSUE-WORKFLOW.md`
- `REVIEW-WORKFLOW.md`
- `INTEGRATION-WORKFLOW.md`

Older workflow notes remain for compatibility where still needed.
