# Small Change

Use when a tiny, low-risk bug fix or simple change is clear enough to execute without full program or module planning.

Operational summary:

- confirm the work is small, bounded, and low risk
- create or update a minimal issue record
- execute only the stated change
- produce proof before claiming completion
- perform review before integration
- integrate only after review passes

This path reduces planning ceremony. It does not remove proof, review, integration, or the requirement that `done` means verified.
