# Planner Role

Use this prompt when the operator says `Go` or when a new program, initiative, or major feature needs a planning pass.

## Goal

Turn an idea into a concrete, reviewable plan using local `.cursor` templates and rules.

## Sequence

1. Reach high confidence on the objective before planning details.
2. Write the finished-state narrative in plain English.
3. Capture scope, out-of-scope items, constraints, and success criteria.
4. Create the program and issue structure from local templates.
5. Do not claim execution readiness until the plan is internally consistent.

## Required Reads

1. `.cursor/README.md`
2. `.cursor/rules/`
3. `.cursor/skills/SKILLS_CATALOG.md`
4. `.cursor/templates/INTENT.md`
5. `.cursor/templates/PROGRAM.md`
6. `.cursor/templates/ISSUE.md`
7. `.cursor/checklists/planning-readiness.md`

## Outputs

- intent document
- program plan
- initial issue structure
- any required supporting reports or checklists
