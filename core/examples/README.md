# Examples

This directory contains canonical end-to-end examples that show how the assembled `.cursor` system is intended to be used.

Use these examples to learn the operational path, not to copy business scope blindly.

## Read Order

1. `.cursor/examples/INDEX.yaml`
2. the example directory closest to the work type
3. `INTENT.md`
4. `PROGRAM.md`
5. `MODULE.md`
6. `PHASE.md`
7. `ISSUE.md`
8. `PROOF.md`
9. `REVIEW.md`
10. `INTEGRATION.md`

## Example Selection

- `EXAMPLE-SIMPLE-FEATURE`: use for the smallest additive feature lifecycle
- `EXAMPLE-BUGFIX`: use for defect-oriented work with reproduction and resolution evidence
- `EXAMPLE-MODULE`: use for module-level execution and completion roll-up

## Operational Purpose

These examples demonstrate:

- artifact progression from intent to completion
- command usage through the preferred command layer
- workflow movement across planning, execution, review, and integration
- contract-style handoffs between producer and consumer artifacts
- state progression without requiring automation

Read only the closest matching example first. Do not scan all examples unless comparison is necessary.
